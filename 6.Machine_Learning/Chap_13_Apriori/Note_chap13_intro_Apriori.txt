#---------------------------------------------------------#
#------------ Apriori Algorithm Explained ----------------#
#---------------------------------------------------------#

The Apriori algorithm is a fundamental algorithm in data mining used for frequent itemset mining and association rule learning. It works on the principle that:

"If an itemset is frequent, then all of its subsets must also be frequent."

After deploying the Apriori, we will know which item is usually bought along with which item
=> But them closer in the shelf, so that customers can see them both and make them want to buy them both
=> Enhance sales

Key Concepts
------------

1. Itemset: A collection of one or more items.


2. Support: How frequently an itemset appears in the dataset.
   Formula:
   	
	Support(A) = (Number of transactions containing A) / (Total number of transactions)



3. Confidence: Likelihood that item B is also bought when item A is bought.
   Formula:

	 Confidence(A ⇒ B) = Support(A ∪ B) / Support(A)



4. Lift: How much more likely B is purchased when A is purchased compared to B being purchased independently.
   Formula:

	 Lift(A ⇒ B) = Confidence(A ⇒ B) / Support(B)

Lift > 1:
    - A and B appear together more often than expected.
    - Positive correlation.
    - The presence of A increases the likelihood of B.
    - Strong association.

Lift = 1:
    - A and B appear together exactly as expected under independence.
    - No correlation.
    - The presence of A has no effect on the likelihood of B.

Lift < 1:
    - A and B appear together less often than expected.
    - Negative correlation.
    - The presence of A decreases the likelihood of B.
    - Weak or inverse relationship.


Apriori Algorithm Steps
-----------------------

1. Set a minimum support threshold.
2. Find all frequent 1-itemsets that meet the minimum support.
3. Generate candidate 2-itemsets from frequent 1-itemsets.
4. Count the support of candidate 2-itemsets.
5. Prune those with support less than the threshold.
6. Repeat the process to generate 3-itemsets, 4-itemsets, etc., until no more frequent itemsets can be found.
7. Generate association rules from the frequent itemsets using the confidence and lift thresholds.


{Bread, Butter} ⇒ Milk
LHS = Left-Hand Side of the rule = {Bread, Butter}
RHS = Right-Hand Side of the rule = Milk

#-------------------------------------------------#
#------------ Example: 2-itemsets ----------------#
#-------------------------------------------------#

Given Transactions:

T1: {Milk, Bread, Butter}
T2: {Bread, Butter}
T3: {Milk, Bread}
T4: {Milk, Butter}
T5: {Bread, Butter}

Total Transactions = 5
Minimum Support Threshold = 0.6 (or 60%)

Step 1: Count Support for 1-Itemsets

- Milk: appears in T1, T3, T4 → 3/5 = 0.6
- Bread: T1, T2, T3, T5 → 4/5 = 0.8
- Butter: T1, T2, T4, T5 → 4/5 = 0.8

All 1-itemsets meet the support threshold.

Step 2: Generate 2-Itemsets and Count Support

- {Milk, Bread} → T1, T3 → 2/5 = 0.4 → discard
- {Milk, Butter} → T1, T4 → 2/5 = 0.4 → discard
- {Bread, Butter} → T1, T2, T5 → 3/5 = 0.6 → keep

Step 3: Generate Association Rules from {Bread, Butter}

Rule: Bread ⇒ Butter
- Support(Bread ∩ Butter) = 3/5 = 0.6
- Support(Bread) = 4/5 = 0.8
- Confidence = 0.6 / 0.8 = 0.75
- Support(Butter) = 4/5 = 0.8
- Lift = 0.75 / 0.8 = 0.9375

Rule: Butter ⇒ Bread
- Confidence = 0.6 / 0.8 = 0.75
- Lift = 0.75 / 0.8 = 0.9375

Conclusion
----------

- The only frequent 2-itemset is {Bread, Butter}.
- Both rules have a confidence of 0.75 and lift of 0.9375.
- Since lift is < 1, this suggests a slight negative correlation — Bread and Butter are bought together, but not more often than chance.



#-------------------------------------------------#
#------------ Example: 3-itemsets ----------------#
#-------------------------------------------------#

Transactions:
-------------
T1: {Milk, Bread, Butter}
T2: {Milk, Bread, Butter}
T3: {Milk, Bread}
T4: {Bread, Butter}
T5: {Milk, Butter}

Total Transactions: 5
Minimum Support Threshold: 0.6 (or 60%)

Step 1: 1-Itemsets with Support
-------------------------------
- Milk: appears in T1, T2, T3, T5 → 4/5 = 0.8 ✅
- Bread: T1, T2, T3, T4 → 4/5 = 0.8 ✅
- Butter: T1, T2, T4, T5 → 4/5 = 0.8 ✅

All 1-itemsets are frequent.

Step 2: 2-Itemsets with Support
-------------------------------
- {Milk, Bread}: T1, T2, T3 → 3/5 = 0.6 ✅
- {Milk, Butter}: T1, T2, T5 → 3/5 = 0.6 ✅
- {Bread, Butter}: T1, T2, T4 → 3/5 = 0.6 ✅

All these 2-itemsets are frequent.

Step 3: 3-Itemsets with Support
-------------------------------
- {Milk, Bread, Butter}: appears in T1, T2 → 2/5 = 0.4 ❌
(does not meet 0.6 threshold)

Let's reduce the support threshold to 0.4 for this example:

Updated Minimum Support Threshold = 0.4 (40%)

Now {Milk, Bread, Butter} is frequent.

Frequent 3-itemset:
--------------------
- {Milk, Bread, Butter} → support = 2/5 = 0.4 ✅

Step 4: Generate Association Rules from 3-Itemset
-------------------------------------------------

Rule 1: {Milk, Bread} ⇒ Butter
- Support({Milk, Bread, Butter}) = 2/5 = 0.4
- Support({Milk, Bread}) = 3/5 = 0.6
- Confidence = 0.4 / 0.6 = 0.67
- Support(Butter) = 4/5 = 0.8
- Lift = 0.67 / 0.8 = 0.8375

Rule 2: {Milk, Butter} ⇒ Bread
- Support({Milk, Bread, Butter}) = 0.4
- Support({Milk, Butter}) = 3/5 = 0.6
- Confidence = 0.4 / 0.6 = 0.67
- Support(Bread) = 0.8
- Lift = 0.67 / 0.8 = 0.8375

Rule 3: {Bread, Butter} ⇒ Milk
- Support({Milk, Bread, Butter}) = 0.4
- Support({Bread, Butter}) = 3/5 = 0.6
- Confidence = 0.4 / 0.6 = 0.67
- Support(Milk) = 0.8
- Lift = 0.67 / 0.8 = 0.8375 

(or Lift {Bread, Butter ⇒ Milk} = Support({Milk, Bread, Butter}) / [Support({Bread, Butter}) * Support(Milk)]
                         	 =              0.4              /  [       0.6               *       0.8    ]  
                                 =                             0.8375                                          )

Interpretation:
---------------
- All rules have confidence = 0.67 (i.e., 67% of the time, the RHS item appears with the LHS items).
- Lift < 1 → there is a slight negative correlation.
- Still, the confidence is decent, so these rules may still be of interest, especially in smaller datasets.

Summary:
--------
- With a support threshold of 0.4, we found the frequent 3-itemset: {Milk, Bread, Butter}.
- From this, we derived 3 association rules with confidence = 0.67 and lift = 0.8375.
